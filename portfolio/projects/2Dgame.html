Jumping Jack - 2D Game

Interactive Browser-Based Platformer Game

Project Overview

Jumping Jack is an engaging 2D platformer game developed using vanilla JavaScript, HTML5, and CSS3. The game challenges players to navigate through increasingly difficult obstacles while maintaining a running sequence. It demonstrates expertise in game development mechanics, animation, collision detection, and interactive web applications.

The game features smooth animations, sound effects, scoring systems, and game-over mechanics - all built from scratch without external game engines.

Technical Implementation

Game Architecture

  • Rendering: Canvas-free DOM-based rendering using HTML elements
  • Animation: RequestAnimationFrame using SetInterval for frame timing
  • Physics: Simple acceleration-based jumping with gravity simulation
  • Collision: Rectangle-based collision detection between player and obstacles
  • State Management: Multiple game states (running, jumping, dead, game over)

Key JavaScript Functions

  • run() - Cycles through running animation frames
  • jump() - Handles vertical movement and jump animation
  • moveBackground() - Creates parallax scrolling effect
  • createBlock() - Generates random obstacles
  • moveBlocks() - Moves obstacles and detects collisions
  • updateScore() - Increments and displays score
  • keyCheck() - Handles keyboard input (Enter to start, Space to jump)
  • dead() - Displays death animation and game over screen

Technology Stack

  • Frontend: HTML5, CSS3, Vanilla JavaScript (ES5)
  • Graphics: PNG sprite sheets for characters and obstacles
  • Audio: MP3 sound effects
  • Animation: CSS and JavaScript timing (100ms frame rate)
  • Browser APIs: DOM manipulation, Audio API, Event listeners

Game Mechanics Details

üèÉ Running System

The character runs continuously when the game is active. The running animation cycles through 8 sprite frames, creating smooth motion. Each frame updates every 100ms, resulting in a frame rate of 10 FPS for the running animation.

‚¨ÜÔ∏è Jumping Physics

When the player presses the spacebar, the character enters jump mode. The jump animation has 12 frames:

  • Frames 1-7: Moving upward (30px per frame)
  • Frames 8-12: Moving downward (30px per frame)
  • Total jump height: 210 pixels
  • Return to ground automatically triggers running mode

üî• Obstacle System

Obstacles spawn continuously with randomized gaps between them. Each obstacle:

  • Has a random gap range of 400-1000 pixels from the previous obstacle
  • Moves from right to left at 20px per frame
  • Collision is detected when obstacle is between 60-140px from left edge AND player is on ground
  • Uses GIF animation for flame effect

üíØ Scoring System

The score increases by 1 point every 250ms during active gameplay. The score is displayed in the top-left corner and updates in real-time. The final score is shown on the game-over screen.
