Engineering Service Building - Integrated Building Management System

Comprehensive IoT Building Automation with Multiple Subsystems | Nov 2025 - Dec 2025

Project Overview

This is a comprehensive Integrated Building Management System developed as a semester project at the University of Ruhuna. The system demonstrates advanced IoT integration combining multiple embedded systems and microcontrollers working seamlessly through a centralized web-based control interface.

The project showcases expertise in embedded systems, hardware-software integration, and IoT communication protocols. It manages five critical building subsystems - lighting, fire detection, access control, surveillance, and elevator operations - all controlled through a unified web dashboard.

Project Timeline: November 2025 - December 2025
Associated with: University of Ruhuna
Team: Shashika Madhusankha, Omindu Sanvida and Bhanuka Prabath

System Subsystems & Features

üí° Lighting Control System

  • Managed directly by ESP32 Dev module
  • LED strips and individual LEDs controlled through relays
  • Fully controllable via web dashboard
  • Real-time status monitoring

üî• Fire Alarm System (Arduino Mega)

  • DHT11 sensors detect fire conditions through temperature & humidity monitoring
  • Automated response: Motor activates water vapor for 30 seconds
  • Alarm triggers for 45 seconds with blinking red LEDs and buzzers
  • Manual web-based trigger override for LEDs and buzzers
  • Feedback LED indicator for system status

üöó Car Park Gate System (Arduino Uno)

  • Two RFID readers for entry and exit access control
  • Servo motor controls 90-degree gate rotation
  • Auto-close functionality: Gate closes after 3 seconds automatically
  • Safety feature: Tag reading disabled while gate is open
  • Manual gate operation available via website

üìπ CCTV Surveillance System

  • ESP32-CAM module provides live video feed
  • Captured images stored on SD card
  • Live feed embedded in the main control website
  • Real-time monitoring capability

üõó Elevator Control System (Arduino Uno)

  • NEMA17 stepper motor for precise movement control
  • Arduino EEPROM stores current cabin position (survives power loss)
  • Accurate floor stops using precise step counting
  • Target floor requests via Serial communication
  • Real-time current floor display

Architecture & Technical Stack

üîå Hardware Architecture

Master Node: ESP32 Dev Module

  • Hosts the web server using WiFiServer
  • Manages direct communication with Arduino Mega and Unos via UART Serial
  • Controls lighting system directly
  • Integrates ESP32-CAM for live video feed

Slave Nodes & Communication

  • Arduino Mega: Fire Alarm System with DHT11 sensors, relays, and motor control
  • Arduino Uno (1): Car Park Gate System with RFID readers and servo motor
  • Arduino Uno (2): Elevator Control with stepper motor and EEPROM
  • ESP32-CAM: CCTV surveillance with SD card storage
  • Communication Protocol: UART Serial for inter-device communication

üíª Software Stack

  • Frontend: HTML, CSS, and JavaScript web interface
  • Backend: Arduino/ESP32 firmware programmed in C++
  • Web Server: WiFiServer on ESP32 Dev module
  • Communication: UART Serial protocol between microcontrollers
  • Storage: Arduino EEPROM, SD card for images

Centralized Control Dashboard

The web-based dashboard provides unified control for all five subsystems. Users can monitor sensor readings in real-time, view live CCTV footage, manually trigger alarms, control lighting, operate gates, and manage elevator operations all from a single interface.

The responsive design works across desktop and mobile devices, enabling facility managers to control the building from anywhere with internet access.
